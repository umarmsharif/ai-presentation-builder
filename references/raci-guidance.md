# RACI Guidance

Sourced from the Zong "Deputy Director Digital Platforms" interview deck session (May 2026). The biggest content error of that session was a RACI using generic business functions instead of the actual org the role sits in.

---

## The rule

**Before drawing any RACI, get the real reporting structure.** If you do not know:
- Who reports directly to this role
- Which sister departments it coordinates with
- Which executive it escalates to

then ask. Do not default to generic functions (Marketing / Sales / Network / Customer Care) — these are placeholders that a real interviewer or client will immediately see through.

---

## Default structure: full A/R/C/I

Always default to the four-column form. Never drop Accountable.

| Column | Meaning | Who it usually is |
|---|---|---|
| **A — Accountable** | Final decision authority; owns the outcome | The role being described (one person/role only) |
| **R — Responsible** | Does the work; executes | Direct reports, sub-team leads |
| **C — Consulted** | Input sought before decisions; two-way | Sister departments, subject matter experts |
| **I — Informed** | Notified after decisions; one-way | Leadership above, downstream functions |

The **Accountable row is the single most important cell on a role-ownership slide.** It answers "who is on the hook?" — which is the primary question any RACI reader is asking.

---

## Common mistakes

- **R/C/I without A:** omitting Accountable makes the grid look like a responsibility matrix, not an ownership model. Always include it.
- **Multiple Accountable entries per row:** exactly one person/role is Accountable per row. If two entries appear, the org has an unresolved ownership gap — surface it, don't paper over it.
- **Generic rows that don't map to the real org:** rows like "Marketing" or "Sales" are valid only if those are actual coordination partners. Replace with the specific team or role name.
- **Too many rows:** a RACI with 12+ rows is a data dump. Aim for 5–8 rows covering the most consequential decisions/workstreams. Group minor activities.

---

## Visual treatment

- INK column headers (`INK` fill, white text, tracked caps) — consistent with the Pattern 2.19 table style.
- Accountable cells: accent-colour fill (`PRIMARY` or `A` amber) to make ownership scannable at a glance.
- Responsible cells: light `PANEL` fill.
- Consulted + Informed cells: white fill, `LINE` border only.
- Row labels in `BODY` weight; decision/workstream text in regular weight.
