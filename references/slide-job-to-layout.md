# Slide Job → Layout Mapping

Sourced from the Zong "Deputy Director Digital Platforms" interview deck session (May 2026). Use this table in Steps 1.5 and 4.5 to make layout selection auditable, not intuitive.

**How to use:** identify the *job* the slide must do for the reader — what decision or understanding are you creating? — then pick the layout that carries that job. The job drives the structure; the structure drives the visual.

---

| Slide job | Primary layout | Notes |
|---|---|---|
| Problem with N supporting proofs | `diagnostic-three-panel` (N=3) or N-panel variant | Each panel is one proof. Titles are findings, not labels. |
| Contrast between two states / options | `two-column-with-delta` | Delta column carries the "so what". Column headers must be mutually exclusive. |
| Parallel options for reader to choose between | `three-col-comparison` or `N-column-cards` | All columns same structure. One row = one evaluation criterion. Never mix criteria across columns. |
| Inputs → transformation → outputs | `source-to-destination` (Pattern 2.20) | Left column = inputs, centre arrow/process = mechanism, right = outcomes. Resist adding a fourth column. |
| Layered system or tech stack | `vertical-stack` | Layers read bottom-up (foundation at bottom). Foundation row bonds to stack — zero gap. |
| Role breakdown / ownership model | `2x2-ownership` or `N-card-grid` | Use 2x2 when two axes organise the roles. Use N-card when roles are parallel with no orthogonal axes. |
| Time-phased plan / workstream sequencing | `phased-roadmap-gantt` | Workstreams are rows; time is horizontal. Milestone strip uses INK-emphasis treatment (not soft-amber). |
| Outcomes under uncertainty / scenarios | `tiered-scenario-columns` | Columns = scenarios (base / upside / downside). Rows = outcome dimensions. Label scenarios at top, not in body. |
| Prioritisation across many items | `prioritisation-matrix` | Two axes = effort vs. impact (or similar). Dots are items. Quadrant labels in muted type, not heavy fills. |
| Single metric or message that anchors the deck | `stat-hero` or `big-numeral-findings` | **Condiment only** — section bridge, not a bulk slide. One number, one sentence explanation. |
| Executive synthesis / answer-first summary | `executive-summary` | Thesis at top (the answer), three to five supporting points below. No new information — only what the deck proves. |

---

## Selection discipline

1. Write the action title first (Step 1.6). If you cannot write the title, you don't know the job yet — do not pick a layout.
2. Match the job to a row above. If two layouts tie, pick the denser one.
3. Only escalate to `AskUserQuestion` if the user has expressed a visual preference earlier in the session, or if the job genuinely does not appear in this table.

## What is NOT a valid job

- "Overview" — this is a topic, not a job. Reframe as a diagnostic job or cut the slide.
- "Introduction to X" — orient in the Context line, not a whole slide.
- "Background" — background is a crutch. State the situation in the executive summary and move on.
