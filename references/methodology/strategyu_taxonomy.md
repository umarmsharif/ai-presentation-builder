# StrategyU consulting presentation framework

Source: https://strategyu.co/consulting-presentations/. Captured 2026-04-26.

## 5-step build process

1. **Define clear objective** — one-sentence outcome statement. Identify document type (live presentation vs. report-heavy deck).
2. **Structure using frameworks** — apply SCQA + Pyramid Principle.
3. **Create presentation** — build outline with standard components (title → exec summary → body → appendix).
4. **Design with simplicity** — minimalist principles (whitespace, action titles, charts over paragraphs).
5. **Practice storytelling** — rehearse delivery and narrative flow.

## SCQA framework

- **Situation** — context the audience already knows
- **Complication** — the problem that breaks the situation
- **Question** — the implicit question raised by the complication
- **Answer** — the data-backed answer

Use SCQA for any deck whose audience is skeptical or unfamiliar with the topic.

## Pyramid Principle (Barbara Minto)

- **Phase 1 bottom-up:** distill findings into one big idea
- **Phase 2 top-down:** structure so audience grasps main ideas immediately, slides logically support core concepts
- Top-down delivery answers SCQA questions

## Ghost deck

Bare-bones outline with only slide titles + minimal bullets. Maps "answers" to hypotheses before any visual design. Use early. Throw away cheaply.

## Action titles

Every slide title states the takeaway, not the topic.
- ❌ "Revenue Analysis"
- ✅ "Revenue Growth Outpaces Market Trends by 15%"

## Standard deck components

- Title slide (topic, name, date, logos)
- Executive summary (high-level recommendation + rationale)
- Body slides (one key insight per slide with data)
- Appendix (backup data + supporting details)

## Design rules

- Word economy — concise bullets, every word supports the message
- Visual hierarchy — fonts, colors, spacing guide the eye
- Whitespace — prevents overload, maintains focus
- Charts over paragraphs (annotated with callouts + source notes)
- One key insight per slide
- Rule of Three — communicate in triads when possible
